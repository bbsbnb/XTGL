# 🏗️ 工程项目证据链管理系统

## 一句话

**用文件夹 + 命名规则 + 一张索引表，解决结算时证据找不到、扯皮的问题。**

---

## 📂 总结构


## 🔧 工具使用

### 方法1️⃣ 双击运行（推荐）

双击 `启动检查工具.bat`，菜单式操作：
- 检查/扫描项目
- 新建项目
- 管理数据库索引
- 导出证据清单

### 方法2️⃣ 命令行

**检查工具（evidence_check.py）：**
```
python 模板与规则/evidence_check.py                  # 检查当前目录
python 模板与规则/evidence_check.py <项目文件夹>      # 检查指定项目
python 模板与规则/evidence_check.py --scan-all       # 扫描所有项目
python 模板与规则/evidence_check.py --init <项目名>   # 新建项目
python 模板与规则/evidence_check.py --export <项目>   # 导出证据清单
```

**数据库索引工具（evidence_db.py）：**
```
python 模板与规则/evidence_db.py init                # 初始化数据库
python 模板与规则/evidence_db.py add                  # 交互式添加记录
python 模板与规则/evidence_db.py list                  # 列出所有记录
python 模板与规则/evidence_db.py search <关键词>      # 搜索
python 模板与规则/evidence_db.py stats                # 统计报告
python 模板与规则/evidence_db.py projects             # 所有项目概况
python 模板与规则/evidence_db.py import <CSV文件>     # 从CSV导入
python 模板与规则/evidence_db.py export               # 导出CSV
python 模板与规则/evidence_db.py info <序号>          # 查看详情
python 模板与规则/evidence_db.py edit <序号>          # 编辑记录
python 模板与规则/evidence_db.py delete <序号>        # 删除记录
```

## 🚀 今天就能开始的 3 步

### 第1步：建项目文件夹

从上面复制一个项目文件夹（如 `_示例项目-2025-001`），重命名为你的项目名，删掉下划线前缀。

### 第2步：开始往里丢文件

拿到什么证据就丢进对应的子文件夹。

**关键规则：** 丢进去之前，按命名规则重命名（见 `01-文件命名规则.md`）。

### 第3步：更新索引表

每丢一个文件，在 `02-证据链总索引模板.csv` 里加一行记录。

## 🤔 这个体系解决了什么

| 问题 | 怎么解决 |
|------|---------|
| 结算时证据找不到 | 统一文件夹结构 + 命名规则，按项目/阶段/类型三级定位 |
| 扯皮（有没有这东西？版本对不对？） | 索引表记录版本号 + 签认状态 + 时间戳，有据可查 |
| 资料太多、整理混乱 | 只有8个固定阶段，每个阶段下只有固定子分类，不乱 |
| 离职交接/人员变动 | 目录结构标准化，谁接手都能看懂 |

---

## 💡 小贴士

- **文件夹结构不要自己乱改** — 统一的结构是检索的基础
- **命名规则是灵魂** — 按规则命名后，Windows搜索、Everything、grep都能秒级定位
- **索引表不是可选项** — 项目完结时，索引表导出就是证据清单，直接用于结算
