# 文件命名规则

> **这是整个体系最关键的一环。** 按规则命名的文件，Windows搜一下就能找到，不需要打开每个文件看。

## 📐 标准格式

```
YYYYMMDD-类别-简述-版本号-状态.扩展名
```

| 段 | 必填 | 说明 | 示例 |
|----|------|------|------|
| YYYYMMDD | ✅ | 文件产生日期 | `20250115` |
| 类别 | ✅ | 缩写代码（见下方） | `VS` |
| 简述 | ✅ | 一句话说清事由 | `主体结构二层梁板钢筋绑扎` |
| 版本号 | ✅ | v1.0 / v2.0 / v1.1 | `v1.0` |
| 状态 | ✅ | 签认状态 | `已签认` / `待确认` / `已否决` |

**示例：**
```
20250115-VS-主体结构二层梁板钢筋绑扎签证-v1.0-已签认.pdf
20250320-BG-裙楼外墙做法变更通知-v2.0-待确认.pdf
20251110-GCL-1号楼标准层混凝土工程量确认-v1.0-已签认.pdf
```

## ⚙️ 版本号规则

| 版本 | 含义 | 示例 |
|------|------|------|
| v1.0 | 初版/首次提交 | 签证单刚签完，初版扫描 |
| v1.1 | 小幅修改 | 补了个签字 |
| v2.0 | 重大修改 | 金额/范围大变 |
| final | 最终版 | 结算前确认的最终版本 |

## 📌 状态代码

| 状态 | 含义 |
|------|------|
| 已签认 | 甲方/监理已签字或盖章确认 |
| 待确认 | 已提交但还没签回来 |
| 已否决 | 对方不认可，有书面证明 |
| 自行归档 | 我方内部记录，不需签认 |

## 🎯 举例说明

**坏的命名（你现在的样子）：**
```
签证单.pdf
新建文档(1).docx
微信图片_20250324145836.jpg
最终版最终版-结算书-真的最终了.xlsx
```

**好的命名（改成这样）：**
```
20250115-VS-主体结构二层梁板钢筋绑扎签证-v1.0-已签认.pdf
20250210-BG-外墙保温材料变更通知-v2.0-待确认.pdf
20250324-PHOTO-YB-1号楼地下室底板防水施工过程.jpg
20251105-JSS-项目整体结算书-v1.0-已签认.xlsx
```

> 改一次命名花 10 秒，结算时找证据省 10 小时。
