# 归档流程规则

## ⏱ 三个时间节点

### 1️⃣ 日常归档（业务发生时立即做）

| 场景 | 责任人 | 做什么 |
|------|--------|--------|
| 收到签证单 | 现场工程师 | 扫描/拍照 → 按命名规则重命名 → 放到 `01-过程签证/签证单/` → 更新索引表 |
| 设计变更下发 | 技术负责人 | 同上，放到 `02-设计变更/变更通知单/` |
| 隐蔽工程验收 | 质检员 | 照片 + 验收记录，放到 `04-验收记录/隐蔽工程验收/` |
| 材料进场 | 材料员 | 到场验收单 + 合格证，放到 `03-施工过程/材料进场验收/` |
| 会议纪要 | 项目秘书 | 参会签到表 + 会议纪要，放到 `06-往来函件/会议签到表/` |
| 往来函件 | 项目秘书 | 放到 `06-往来函件/甲方来函/` 或 `06-往来函件/我方发函/` |

**原则：** 证据在手里不过夜，当天拿到当天归档。

### 2️⃣ 周/月检查（保持体系不崩）

每周末或每月初花 10 分钟：

- [ ] 检查本周产生的证据是否都已归档
- [ ] 检查文件名是否都按规则
- [ ] 检查索引表是否更新
- [ ] 检查有没有遗漏的签认（签字/盖章）

### 3️⃣ 结算前整理（项目收尾）

计划结算前 2 周，做一次全面清点：

- [ ] 打开 `03-结算核对清单.md` 逐项对照
- [ ] 找出所有缺项，列出缺项清单
- [ ] 对缺项发起补签/补录流程
- [ ] 补全索引表
- [ ] 导出索引表作为最终证据清单

## 📋 归档标准

### 文件格式

| 类型 | 推荐格式 | 说明 |
|------|---------|------|
| 纸质文件扫描件 | PDF（推荐） | 单页用JPG，多页合并为PDF |
| 电子文件原件 | 原格式 | Word/Excel/PDF 保持不变 |
| 照片 | JPG | 建议 2-5MB，不要压缩到看不清字 |
| 聊天记录/邮件 | PDF 或 截图 | 关键对话截图保存 |

### 每份证据需要包含的信息

1. **时间** — 文件产生日期（不是归档日期）
2. **事由** — 是什么事（一句话）
3. **签认方** — 谁签字确认了（甲方/监理/我方）
4. **状态** — 已签认 / 待确认 / 已否决
5. **版本** — 第几次修改

> 以上信息统一通过**命名规则** + **索引表** 来承载，不需要额外加备注。

---

## 🚫 禁止行为

- ❌ 文件名写 "新建文档(1).pdf"、"微信图片_20250324145836.jpg"
- ❌ 文件随手丢桌面、微信收藏、邮箱附件，不归档
- ❌ 多个版本混在一起，不标版本号
- ❌ 索引表不更新
- ❌ 文件夹自己加新分类（确实需要新分类时，先和团队商量统一规则）
